# 推栏AI中枢管理后台 - 用户指南

## 概述

推栏AI中枢管理后台是一个AI增强型控制系统，用于智能化管理从需求明确、任务下发到落地实施的全链路。本系统整合了项目管理、知识库管理、多AI模型调度、工作流自动化、Skill调用以及历史记忆与上下文处理。

## 系统架构

系统基于Phase 1-4完整设计方案实现：

- **Phase 1**: 基础框架 - 项目管理仪表盘、知识库浏览器、基础AI需求分析
- **Phase 2**: AI增强 - 多模型集成与调度、工作流编排引擎、Skill仓库与执行器
- **Phase 3**: 记忆系统 - 历史记忆存储与检索、上下文管理系统、智能推荐
- **Phase 4**: 高级功能 - 高级工作流设计器、团队协作、分析与报告系统

### 2. 主要功能模块

#### 仪表盘
- 项目进度概览
- AI模型使用统计
- 系统状态监控
- 最近活动跟踪

#### 项目管理
- **版本管理**: 查看各版本进度、状态和时间线
- **需求看板**: 管理功能需求，按优先级排序
- **任务甘特图**: 可视化任务进度和依赖关系
- **交互功能**: 点击需求查看详情，创建新版本

#### 知识库中枢
- **知识图谱**: 可视化知识关联关系
- **页面浏览**: 查看知识库页面，按类型筛选
- **搜索功能**: 语义搜索知识库内容
- **分类统计**: 按类型统计知识库内容

#### AI调度中心
- **模型管理**: 查看AI模型状态和使用情况
- **计划生成器**: 输入需求描述，AI生成实施计划
- **任务分解**: AI将复杂任务分解为可执行子任务
- **代理调度**: 为任务分配最适合的AI模型

#### 工作流编排
- **可视化设计器**: 拖拽节点构建工作流
- **节点库**: 预置AI任务、Skill调用、条件分支等节点
- **工作流列表**: 管理现有工作流，查看执行状态
- **执行历史**: 查看工作流执行日志和结果

#### Skill仓库
- **Skill分类**: 按功能分类浏览Skills
- **热门Skills**: 查看最常用的Skills
- **使用统计**: 可视化Skill使用频率
- **导入/创建**: 导入现有Skill或创建新Skill

#### 记忆系统
- **记忆存储**: 记录项目经验、技术方案、问题解决方法
- **语义搜索**: 基于自然语言检索相关记忆
- **关联推荐**: 智能推荐相关记忆和知识
- **类型分布**: 统计各类记忆的分布情况

#### 上下文管理
- **上下文容器**: 为任务创建上下文，包含背景信息和约束条件
- **版本控制**: 管理上下文版本和变更历史
- **上下文注入**: 将上下文注入到AI任务中
- **合并功能**: 合并多个相关上下文

## 交互功能演示

### 项目管理示例
1. 点击左侧导航栏的"项目管理"
2. 查看版本卡片，点击"V2.8规划中"查看详情
3. 在需求看板中点击任意需求查看详细描述
4. 点击"创建新版本"按钮体验版本创建流程
5. 查看甘特图了解任务时间线

### AI调度示例
1. 点击左侧导航栏的"AI调度中心"
2. 查看各AI模型的状态和使用情况
3. 在计划生成器中输入："优化推栏小程序聊天界面，增加消息撤回功能"
4. 点击"生成实施计划"按钮
5. 查看AI生成的详细实施计划

1. 点击左侧导航栏的"工作流编排"
2. 点击"添加节点"按钮
3. 从节点库拖拽"AI任务"、"Skill调用"、"条件分支"节点到画布
4. 点击"保存工作流"按钮
5. 点击"运行工作流"按钮体验工作流执行

## 技术特点

### 前端技术
- **响应式设计**: 适配桌面端显示
- **数据可视化**: 使用Chart.js展示统计图表
- **交互式UI**: 支持拖拽、点击、表单等交互
- **模块化架构**: 各功能模块独立，易于扩展

### AI集成
- **多模型支持**: Claude、GPT、GLM、DeepSeek等
- **智能调度**: 根据任务类型自动选择最佳模型
- **上下文感知**: 基于历史记忆和当前上下文优化输出
- **自动化执行**: AI代理自动执行分解后的任务

### 知识管理
- **Obsidian集成**: 与现有知识库无缝连接
- **语义关联**: 自动建立知识间的语义关联
- **版本追踪**: 记录功能模块的版本演进历史
- **智能推荐**: 基于当前任务推荐相关知识

## 扩展开发

### 添加新模块
1. 在`frontend/index.html`中添加新的`<section>`
2. 在`frontend/js/modules.js`中扩展`ModuleExtensions`类
3. 在`CenterSystemApp`类中添加对应的渲染方法
4. 添加CSS样式到`frontend/css/styles.css`

### 集成后端API
1. 在`backend/`目录创建API服务
2. 修改前端JavaScript调用真实API
3. 实现数据持久化和业务逻辑
4. 配置数据库和缓存服务

### 添加新Skills
1. 参考现有的Hermes Agent Skills格式
2. 定义Skill的触发条件和执行步骤
3. 在Skill仓库中注册新Skill
4. 在工作流中集成新Skill

## 故障排除

### 常见问题

**Q: 页面无法加载**
A: 检查端口8000是否被占用，尝试修改端口号

**Q: 图表不显示**
A: 检查网络连接，确保Chart.js库正常加载

**Q: 交互功能无响应**
A: 检查浏览器控制台是否有JavaScript错误

**Q: 数据不更新**
A: 当前为演示版本，数据为模拟数据。集成后端API后可实现真实数据
